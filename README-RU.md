# 🧠 Nehil (נְחִיל)

**Децентрализованная нейросеть поверх mesh-сетей. Никакого облака. Никакого центрального сервера. Ни единой точки отказа.**

Nehil — это рой устройств (телефоны, Raspberry Pi, ноутбуки), которые совместно обучают и запускают AI-модели без интернета и без центрального координатора. Узлы общаются только через локальную mesh-сеть (Wi-Fi Direct, Bluetooth, LoRa или обычный Ethernet) и обмениваются **знаниями**, а не сырыми данными.

## 🤔 Зачем это нужно

**Современный AI централизован:**
- Твои данные уходят с устройства
- Нужен постоянный интернет
- Одна компания контролирует модель
- Нет интернета — нет AI

**Nehil переворачивает модель:**
- Данные никогда не покидают устройство
- Работает офлайн, где угодно
- Никакого центрального управления
- Сеть сама себе сервер

---

## ⚙️ Как это работает

> **Шаг 1 — Локальное обучение**
> Каждое устройство обучает свою копию модели на локальных данных (фото, показания датчиков, текст). Сырые данные никогда не покидают устройство.

> **Шаг 2 — Сжатие знаний**
> Изменения модели (градиенты или веса) сжимаются в крошечные патчи размером в килобайты, а не мегабайты.

> **Шаг 3 — Gossip-протокол**
> Устройства отправляют патчи соседям через mesh-сеть. Никакого центрального сервера — только peer-to-peer.

> **Шаг 4 — Агрегация**
> Каждое устройство усредняет свою модель с полученными патчами. Знания распространяются как слух.

> **Шаг 5 — Повтор**
> После достаточного числа раундов все устройства приходят к одной умной модели — обученной коллективно, без доступа к чужим данным.

---

## ✨ Что делает Nehil уникальным

- **Не нужен интернет** — работает в лесу, подвале, подводной лодке
- **Нет центрального сервера** — нет единой точки отказа и контроля
- **Работает на LoRa** (биты в секунду) — адаптируется к сверхнизкой пропускной способности
- **Данные остаются на устройстве** — приватность по умолчанию
- **Устойчивость к потере узлов** — сеть самовосстанавливается

## 🎯 Сценарии использования

**Офлайн-умный дом**
Датчики изучают твои привычки, не отправляя данные в Китай или AWS.

**Поисково-спасательные работы**
Команда в глуши без сотовой связи — телефоны совместно распознают звук вертолёта и передают координаты через mesh-сеть.

**Анонимное коллективное обучение**
Журналисты обучают модель распознавания цензуры, не раскрывая свои документы.

**Критическая инфраструктура**
Подводная лодка, шахта или арктическая станция — интернета нет, но внутренняя mesh-сеть держит AI на ходу.

**Физическое тестирование на проникновение**
Пять Raspberry Pi в рюкзаке работают как распределённый AI-ассистент: анализируют Wi-Fi, распознают устройства, предлагают векторы атак. Никакого облака, никаких логов на серверах клиента.

## 📊 Статус проекта

⚠️ **Proof of Concept** — есть работающая эмуляция трёх узлов на одном ноутбуке через Docker. Тесты на реальном железе (Raspberry Pi, LoRa) — следующий этап.

**Готово:**
- Базовая связь между узлами (HTTP)
- Каркас обмена моделями
- Docker Compose для трёх узлов

**В планах:**
- Федеративное усреднение (FedAvg)
- Сжатие градиентов
- Транспорт через Bluetooth и Wi-Fi Direct
- Поддержка LoRa
- Инструкция по развёртыванию на Raspberry Pi

## 🚀 Быстрый старт (эмуляция трёх узлов)

**Способ 1: Docker (самый простой)**
```bash
git clone https://github.com/yourusername/nehil.git
cd nehil
docker-compose up --build
```

**Способ 2: Вручную**
```bash
git clone https://github.com/yourusername/nehil.git
cd nehil
pip install -r requirements.txt
python node.py --port 8001 --peers localhost:8002,localhost:8003
```
