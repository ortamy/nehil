# 🧠 Nehil (נְחִיל)

**Децентрализованный протокол для коммуникации нейросетей. Человек — узел. Знание — наследие. Связь — без посредников.**

Nehil — это не платформа. Это способ, которым нейросети разных людей общаются друг с другом напрямую. Через интернет, если он есть. Через mesh-сеть, если его нет. Без облака, без сервера, без единого центра.

Каждое устройство — это самостоятельный разум. Вместе они образуют рой, который учится, делится знаниями и помогает каждому участнику. Как поток (נַחַל), который несёт воду каждому ростку. Как наследие (נַחֲלָה), которое принадлежит всем.

## 🤔 Зачем это нужно

**Современный AI отбирает наследие:**
- Твои данные уходят корпорациям — ты теряешь контроль над своим знанием
- Нужен постоянный интернет — ты зависишь от инфраструктуры, которую не контролируешь
- Одна точка отказа — выключи сервер, и разум исчез
- Модели принадлежат компаниям, а не людям

**Nehil возвращает наследие человеку:**
- Данные никогда не покидают устройство — твоя мудрость остаётся твоей
- Работает офлайн, где угодно — связь через mesh, как ручей находит путь
- Никакого центрального управления — рой самоорганизуется
- Каждый узел — полноправный участник сети знаний

## ⚙️ Как это работает

> **Шаг 1 — Локальное обучение**
> Каждый человек использует свою нейросеть на своём устройстве. Она обучается на его данных, запросах, контексте. Это личный поток знаний.

> **Шаг 2 — Сжатие знаний**
> Изменения модели сжимаются в патчи — маленькие пакеты мудрости. Не данные, а выводы. Не информация, а понимание.

> **Шаг 3 — Gossip-протокол**
> Устройства обмениваются патчами с соседями. Через интернет, если есть. Через Wi-Fi Direct, Bluetooth или LoRa, если нет. Знание течёт как вода — находя путь.

> **Шаг 4 — Агрегация**
> Каждое устройство усредняет свою модель с полученными патчами. Локальное становится общим. Общее обогащает локальное.

> **Шаг 5 — Повтор**
> После достаточного числа циклов сеть приходит к коллективному пониманию. Без доступа к чужим данным. Без центра. Без контролёра.

## 🌊 Дух проекта: что значит נְחִיל

Слово נְחִיל имеет три смысла, и все три заложены в этот проект:

**נַחַל (поток)**
Знания текут от человека к человеку, как вода в ручье. Сеть адаптируется к ландшафту — интернет, mesh, офлайн. Поток находит путь.

**נַחֲלָה (наследие)**
Каждый человек — носитель уникального опыта. Nehil сохраняет это наследие у владельца, но позволяет делиться выводами. Знание принадлежит тому, кто его создал.

**נְחִיל (рой)**
Пчёлы движутся как единый организм без вожака. Узлы Nehil координируются без сервера. Коллективный разум возникает из простых правил взаимодействия.

---

## ✨ Что делает Nehil уникальным

- **Не нужен интернет** — работает в лесу, в поле, в дороге
- **Нет центрального сервера** — нет единой точки отказа и контроля
- **Работает на LoRa** (биты в секунду) — знание течёт даже по тончайшему руслу
- **Данные остаются на устройстве** — наследие не отчуждается
- **Устойчивость к потере узлов** — рой перестраивается, ручей находит новый путь
- **Человек → человек** — это коммуникация нейросетей, принадлежащих людям, а не платформа

## 🎯 Сценарии использования

**Повседневная автономность**
Твоя нейросеть общается с нейросетями друзей напрямую. Нет интернета — mesh-сеть. Нет mesh — Bluetooth. Знания распределены, а не заперты на сервере.

**Поисково-спасательные работы**
Группа в глуши без сотовой связи. Нейросети на телефонах совместно анализируют звуки, изображения, координаты и передают выводы через mesh-сеть.

**Коллективное обучение без риска**
Журналисты, исследователи, активисты совместно обучают модель на чувствительных данных. Данные не покидают устройства — передаются только выводы.

**Критическая инфраструктура**
Подводная лодка, полярная станция, убежище. Внутренняя mesh-сеть держит коллективный AI на ходу без связи с внешним миром.

**Сохранение наследия**
Семья хранит общую нейросеть, обученную на историях, фотографиях и знаниях старших. Наследие (נַחֲלָה) живёт на устройствах близких, а не в облаке корпорации.

## 📊 Статус проекта

⚠️ **Proof of Concept** — работает эмуляция трёх узлов на одном ноутбуке через Docker. Тесты на реальном железе (Raspberry Pi, LoRa) — следующий этап.

**Готово:**
- Базовая связь между узлами (HTTP)
- Каркас обмена патчами знаний
- Docker Compose для трёх узлов

**В планах:**
- Федеративное усреднение (FedAvg)
- Сжатие градиентов для низкой пропускной способности
- Транспорт через Bluetooth и Wi-Fi Direct
- Поддержка LoRa (настоящий mesh)
- Инструкция по развёртыванию на Raspberry Pi
- API для подключения локальных нейросетей
